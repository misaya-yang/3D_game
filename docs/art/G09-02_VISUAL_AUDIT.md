# G09-02 角色与三图视觉审计

> 审计日期：2026-07-16  
> 视口：1920×1080 macOS Release Player  
> 比较原则：同一出生点、上午 10 点、默认第三人称镜头；角色近景使用同一 Player 内的固定证据镜头。
> 历史说明：本页记录第一轮签核事实；当前主角、NPC、人形敌和石将军已由
> [`G09-07_CHARACTER_AUDIT.md`](G09-07_CHARACTER_AUDIT.md) 的第二轮资产取代。

## 结论

G09-02 将 G08-03 的“完整低模占位”收敛成三套可识别主题，并关闭了真实 Player 中会直接影响可玩性的相机遮挡、苍梧导航失败和材质失效问题。改造没有修改任务/敌人数值、内容 ID、存档 Schema 或灰盒碰撞。

主角现使用年轻、束发、持剑并带红金衣摆的 Warrior 低模，通过青灰金色域弱化西式板甲感；NPC 继续以职业轮廓区分，狼/精英狼改为真实四足模型，石将军改为独立巨型怪物模型。三图分别稳定为暖绿村落水岸、冷青山林云雾和低饱和洞窟遗迹。

## 基线问题与处理

| 范围 | 改造前 P0/P1 问题 | 处理结果 |
|------|------------------|----------|
| 主角 | 老年 Monk 替身、正背比例不稳、近景有粗糙程序块面 | 替换为 Warrior；保留发髻、持剑、红金衣摆；统一身高 1.82 m、URP 材质和近景/远景证据 |
| NPC/敌人 | 狼和石将军为程序组合几何；普通/精英/BOSS 远景轮廓接近 | Wolf 与 Goleling_Evolved 接入；精英狼使用更大体型和褐红阵营色；人形敌人沿用 Rogue |
| 青石村 | 荧光青树木、桃色岩石、纯色平地、道路与水岸层级弱 | 草土 1K 地表、两段道路、溪流/木桥、村门/民居/药田/营地；岩石改冷灰石材分级 |
| 苍梧山 | 大门压住镜头、山石过曝、地表无层次、敌人偶发无法创建 NavMesh agent | 缩小并后移山门；林地纹理、冷青雾、松林/竹林/洞府/雷台构图；补运行时 NavMesh surface |
| 黑风洞 | 近黑背景、地面与墙体粘连、房间空洞 | 岩地纹理、五层模块房间、门/遗迹/双火盆照明；提高最低环境亮度并保留压迫感 |
| 相机/材质 | 透明度写入对 URP Opaque 遮挡物无效；少量导入材质可落到 InternalErrorShader | Opaque 遮挡物改用 `forceRenderingOff` 并在离开视线后恢复；场景收尾扫描修复空/不支持材质 |
| 天气/昼夜 | 晴天会把地图基础雾清零；同一套环境光削弱三图差异 | 晴天恢复场景基础雾；昼夜系统读取每图太阳、环境光和雾基线 |

## 运行时完整性

- 新视觉统一挂在 `BudgetVisual` 子节点，导入碰撞会被移除；玩法根碰撞保留。
- 苍梧山以物理碰撞源构建运行时 NavMesh，视觉装饰保持无碰撞，不改变可走路线。
- `BudgetArtCatalog.RequiredRefinementResources` 和 `G0902PlayModeTests` 验证关键模型/贴图确实随 Player 打包。
- 资产来源、许可证、选用文件和 SHA-256 见 [`THIRD_PARTY_ASSETS.md`](THIRD_PARTY_ASSETS.md)。

## 保留边界

本 Goal 不包含原创 4K 雕刻、完整动作重定向、付费资产或地图玩法重设计。低模角色仍共享既有占位动画能力；下一轮若追求接近概念设定图的脸部、衣褶和发丝，应以同骨架正式角色资产替换，而不是继续堆叠程序化几何。
